# M-FE-07: Model Tree Panel（模型树面板）

> **详细设计**: §3.7
> **依赖**: M-FE-08（State Store — semanticModel）
> **目标**: 左侧底部/可切换的树形导航，展示语义模型层级，支持选择、右键操作

---

## 任务清单

### 1. 树组件

- [ ] **1.1** 实现 `ModelTreePanel` React 组件：使用 Ant Design Tree 组件
- [ ] **1.2** 实现 `buildTree(semanticModel): ModelTreeNode[]`：从语义元素构建树节点
  - 找出根元素（ownerId === null）
  - 递归收集子元素（child.ownerId === parent.id）
  - Package 元素显示在普通元素之前
- [ ] **1.3** 每个节点显示：类型图标 + 名称 + 类型后缀（如 "Engine [PartDef]"）
- [ ] **1.4** 测试：创建 3 层嵌套的 PartDef，树形层次正确

### 2. 导航与选择

- [ ] **2.1** 单击树节点 → `Store.selectElements([id])` → 画布上对应元素高亮
- [ ] **2.2** 双击树节点 → 选中元素 + 画布定位到该元素（pan + zoom 到元素位置）
- [ ] **2.3** 画布上选中元素 → 树上对应节点高亮（双向同步）
- [ ] **2.4** 测试：单击树 → 画布高亮；双击树 → 画布中心定位到元素

### 3. 右键菜单

- [ ] **3.1** 实现右键上下文菜单（Ant Design Dropdown）：
  - 删除元素 → 级联删除子元素和关系
  - 重命名 → 内联编辑
  - 新建子元素 → 弹出子菜单（Part/Port/Attribute/Requirement...）
  - 在图定位 → 切换到包含该元素的 Diagram
- [ ] **3.2** 不同元素类型显示不同菜单项（如 Requirement 可添加 Satisfy 关系）
- [ ] **3.3** 测试：右键 Package → 新建子元素 → 选择 PartDef → 创建成功

### 4. 拖拽重组

- [ ] **4.1** 实现树节点拖拽：拖 A 到 B 上 → A.ownerId = B.id（移动）
- [ ] **4.2** 拖拽到 Package 上 → 放入 Package
- [ ] **4.3** 不允许拖到自己的子孙上（防止循环引用）
- [ ] **4.4** 拖拽完成 → `Store.moveElement(elementId, newOwnerId)`
- [ ] **4.5** 测试：拖 PartDef A 到 Package P 上 → A 的 ownerId 变为 P 的 id

### 5. 搜索过滤

- [ ] **5.1** 顶部搜索框：输入文字 → 树实时过滤
- [ ] **5.2** 匹配逻辑：名称包含关键词 OR 限定名包含关键词
- [ ] **5.3** 过滤后自动展开所有包含匹配项的路径
- [ ] **5.4** 清空搜索 → 恢复完整的树（保持之前的展开状态）
- [ ] **5.5** 测试：搜索 "Engine" → 只显示名称含 "Engine" 的节点及其祖先路径

### 6. 状态同步

- [ ] **6.1** Store 中元素增删改时，树自动更新（Zustand subscribe）
- [ ] **6.2** 展开/折叠状态在数据更新后尽量保持
- [ ] **6.3** 测试：通过 Property Panel 修改元素名称 → 树上节点标签同步更新
